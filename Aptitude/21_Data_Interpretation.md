## üìö Part 1: Types of Data Representation

### Tables

Numerical data organized in rows and columns.

**Example**:
| Year | Sales (‚Çπ Cr) | Profit (‚Çπ Cr) |
|------|--------------|---------------|
| 2019 | 150 | 30 |
| 2020 | 180 | 35 |
| 2021 | 220 | 45 |

**Reading Tips**:
- Identify row/column headers
- Note units (thousands, lakhs, crores)
- Look for trends and patterns

---

### Bar Graphs

Rectangular bars representing data magnitudes.

**Types**:
- **Simple bar**: One data series
- **Grouped bar**: Multiple series side by side
- **Stacked bar**: Components stacked on each other

**Reading Tips**:
- Check axis scales carefully
- Compare bar heights/lengths
- Note if scale starts at 0 (truncated scales can mislead)

---

### Line Graphs

Data points connected by lines, showing trends over time.

**Reading Tips**:
- Slope indicates rate of change
- Steeper slope = faster change
- Look for turning points, maxima, minima

---

### Pie Charts

Circular chart divided into sectors representing proportions.

**Key Formula**:
```
Sector angle = (Value/Total) √ó 360¬∞
Percentage = (Sector angle/360¬∞) √ó 100%
```

**Reading Tips**:
- Larger sector = higher proportion
- All sectors sum to 100% or 360¬∞
- Compare relative sizes

---

### Mixed/Combination Charts

Multiple chart types combined.

**Example**: Bar graph with a line overlay showing trend.

## üìê Essential Calculation Skills

### Percentage Calculations

```
% of a value = (Percentage/100) √ó Value
% change = ((New - Old)/Old) √ó 100
```

**Quick Approximations**:
```
10% of X = X/10
5% of X = X/20
1% of X = X/100
25% of X = X/4
50% of X = X/2
```

---

### Ratio Calculations

```
Ratio A:B means A/B = ratio
If A:B = 3:4 and B = 80, then A = 80 √ó (3/4) = 60
```

---

### Average

```
Average = Sum of values / Number of values
```

---

### Growth Rate

**Simple Growth**:
```
Growth rate = ((Final - Initial)/Initial) √ó 100%
```

**CAGR (Compound Annual Growth Rate)**:
```
CAGR = [(Final/Initial)^(1/n) - 1] √ó 100%
```

## üìà Data Analysis Strategies

### Strategy 1: Understand the Data First

Before answering questions:
1. Read all labels, titles, legends
2. Understand units (thousands, lakhs, etc.)
3. Note the time period or categories
4. Identify any data gaps or anomalies

---

### Strategy 2: Estimate Before Calculate

Quick approximation helps:
1. Avoid silly mistakes
2. Choose between close options
3. Save time on complex calculations

---

### Strategy 3: Use Options Wisely

If options are far apart:
- Approximation sufficient
- Work backwards from options

If options are close:
- Careful calculation needed
- Double-check arithmetic

---

### Strategy 4: Look for Patterns

Common patterns in data:
- Increasing/decreasing trends
- Seasonal variations
- Proportional relationships
- Outliers

## ‚ö†Ô∏è Common Traps & Pitfalls

### Trap 1: Unit Confusion
```
Values in lakhs ‚â† Values in crores
Always check units before calculating
```

### Trap 2: Truncated Y-Axis
```
Bar graph not starting from 0
Makes small differences look large
```

### Trap 3: Percentage OF vs Percentage Point
```
10% to 15% is a 5 percentage POINT increase
But it's a 50% increase (5/10 √ó 100)
```

### Trap 4: Base Year Confusion
```
"Increase over previous year" ‚â† "Increase over first year"
Check what the base is
```

### Trap 5: Missing Data
```
Some questions may have incomplete data
May need to use multiple sources
```

## üìä Complex Data Sets

### Caselets

Multiple paragraphs of text containing numerical data.

**Approach**:
1. Read completely first
2. Note key numbers and relationships
3. Organize in a table or diagram

### Multiple Interrelated Tables/Charts

Data spread across multiple representations.

**Approach**:
1. Understand the link between representations
2. Identify which data comes from where
3. Cross-reference carefully
4. Double-check calculations
